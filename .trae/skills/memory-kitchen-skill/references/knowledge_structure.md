# 知识库结构和组织方式

## 1. 知识库位置

知识库统一存储在用户目录下的 `.trae-cn/knowledge` 目录中：

- **Windows**: `C:\Users\Administrator\.trae-cn\knowledge`
- **Linux/Mac**: `~/.trae-cn/knowledge`

## 2. 整体结构

知识库采用层次化的目录结构，便于知识的组织和管理：

```
~/.trae-cn/knowledge/        # 知识库根目录（用户目录下）
  categories/                # 分类目录
    技术知识/                # 技术知识
      前端开发/              # 前端开发
      后端开发/              # 后端开发
      云原生/                # 云原生
      数据库/                # 数据库
      版本控制/              # 版本控制
    项目知识/                # 项目知识
    问题解决方案/            # 问题解决方案
      教程/                  # 教程
      故障排除/              # 故障排除
    最佳实践/                # 最佳实践
  tags/                      # 标签索引
  index.md                   # 知识库索引
  recent-updates.md          # 最近更新
```

### 2.1 跨项目共享
由于知识库存储在用户目录下，所有项目都可以访问和共享同一个知识库，实现知识的积累和复用。

## 2. 分类体系

### 2.1 技术知识
- **前端开发**：前端框架、UI 库、前端工具等
- **后端开发**：后端语言、框架、架构等
- **云原生**：容器、编排、云服务等
- **数据库**：关系型数据库、NoSQL、缓存等
- **版本控制**：Git、代码管理等

### 2.2 项目知识
- **项目管理**：项目规划、进度管理等
- **需求分析**：需求收集、分析等
- **文档管理**：项目文档、技术文档等

### 2.3 问题解决方案
- **教程**： step-by-step 指南
- **故障排除**：常见问题及解决方案

### 2.4 最佳实践
- **编码规范**：代码风格、命名规范等
- **架构设计**：系统架构、设计模式等
- **性能优化**：性能调优、负载测试等

## 3. 知识条目结构

每个知识条目采用 Markdown 格式，包含以下结构：

### 3.1 YAML Frontmatter
```yaml
id: unique-id
 title: 知识标题
tags: [标签1, 标签2]
categories: [分类1, 分类2]
created: 2026-03-04T11:34:56
updated: 2026-03-04T11:34:56
```

### 3.2 正文结构
```markdown
# 知识标题

## 答案
知识内容...

## 标签
- 标签1
- 标签2

## 分类
- 分类1
- 分类2

## 创建时间
2026-03-04T11:34:56

## 更新时间
2026-03-04T11:34:56

## 状态（可选）
- 正常
- 过时
- 需要验证
```

## 4. 索引和导航

### 4.1 知识库索引
`index.md` 文件包含所有知识条目的索引，按分类组织，便于快速定位知识。

### 4.2 最近更新
`recent-updates.md` 文件记录最近更新的知识条目，便于跟踪知识库的变化。

### 4.3 标签索引
`tags/` 目录下按标签组织知识条目，便于按标签检索。

## 5. 命名规范

### 5.1 文件命名
- 使用小写字母和下划线
- 避免使用特殊字符
- 文件名应能反映知识内容

### 5.2 目录命名
- 使用小写字母和连字符
- 目录名应能反映分类内容

## 6. 版本管理

- 使用 Git 进行版本控制
- 定期提交知识库变更
- 保留知识库的历史版本

## 7. 备份策略

- 定期备份知识库目录
- 可以使用云存储进行备份
- 确保备份的安全性和可恢复性

## 8. 扩展和定制

### 8.1 自定义分类
- 可以根据项目需求添加自定义分类
- 分类层次不宜过深，建议不超过 3 层

### 8.2 自定义标签
- 可以根据需要添加自定义标签
- 标签应具有描述性和一致性

### 8.3 自定义字段
- 可以根据需要添加自定义字段到知识条目
- 确保字段的一致性和可扩展性
