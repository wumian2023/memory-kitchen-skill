# 自动记忆知识库技能 - 使用指南

## 概述

自动记忆知识库技能可以自动从对话中收集有价值的信息，并将其存储到知识库中，实现知识的自动化积累和智能化应用。

## 快速开始

### 1. 自动收集当前对话知识

在 Trae 对话中，您可以直接要求 AI 收集当前对话中的知识：

```
请将我们刚才的对话内容收集到知识库中
```

或者：

```
使用 memory-kitchen-skill 收集当前对话的知识
```

### 2. 手动收集指定内容

如果您想收集特定的对话内容，可以：

```
请将以下内容收集到知识库中：
[您的对话内容]
```

### 3. 检索已存储的知识

```
从知识库中检索关于 Git 代理配置的信息
```

或者：

```
我之前存储的关于 Python 内存泄漏的知识在哪里？
```

## 使用场景

### 场景 1：技术问题解决

**对话示例**：
```
用户：如何解决 Git 提交时的代理配置问题？
助手：Git 提交时的代理配置问题通常是由于 Git 配置了错误的代理设置导致的...
用户：请将这个解决方案收集到知识库中
助手：已将 Git 代理配置解决方案存储到知识库中
```

### 场景 2：学习资料整理

**对话示例**：
```
用户：请解释一下 React 18 的并发特性
助手：React 18 引入了并发渲染特性，包括自动批处理、Suspense 改进等...
用户：将这些 React 18 的知识点收集到知识库中
助手：已将 React 18 相关知识整理并存储到知识库
```

### 场景 3：项目知识积累

**对话示例**：
```
用户：我们的项目使用了哪些设计模式？
助手：根据代码分析，项目主要使用了单例模式、工厂模式、观察者模式等...
用户：请将这些项目架构知识收集到知识库
助手：已将项目架构相关知识存储到知识库
```

## 自动化配置

### 方法 1：在个人规则中配置

在您的个人规则文件中添加以下内容，实现每次对话结束后自动收集知识：

```markdown
## 自动知识收集

每次重要对话结束后，自动调用 memory-kitchen-skill 收集对话中的有价值信息。

触发条件：
- 涉及技术解决方案的对话
- 涉及最佳实践的对话
- 涉及问题排查的对话
- 用户明确要求收集的对话
```

### 方法 2：使用环境变量

设置环境变量启用自动收集：

```bash
# Windows PowerShell
$env:MEMORY_KITCHEN_AUTO_COLLECT = "true"

# Linux/Mac
export MEMORY_KITCHEN_AUTO_COLLECT=true
```

## 知识库管理

### 查看知识库

知识库存储位置：
- **Trae-CN**: `C:\Users\Administrator\.trae-cn\knowledge`
- **其他工具**: 对应的配置目录下的 `knowledge` 文件夹

### 知识库结构

```
~/.{ai-tool}/knowledge/
  categories/
    技术知识/
      前端开发/
      后端开发/
      版本控制/
    问题解决方案/
      教程/
      故障排除/
    最佳实践/
  tags/
  index.md
  recent-updates.md
```

### 手动管理知识

您可以直接编辑知识库中的 Markdown 文件，添加、修改或删除知识条目。

## 高级功能

### 1. 批量收集知识

如果您有多个对话文件需要收集：

```bash
python .trae/skills/memory-kitchen-skill/scripts/auto_collect.py path/to/dialogue1.txt
python .trae/skills/memory-kitchen-skill/scripts/auto_collect.py path/to/dialogue2.txt
```

### 2. 自定义分类规则

编辑 `knowledge_organizer.py` 中的分类规则，根据您的需求调整知识分类逻辑。

### 3. 知识检索优化

使用标签和分类进行更精确的检索：

```
检索标签为 "Git" 和 "代理" 的知识
检索分类为 "技术知识 > 版本控制" 的知识
```

## 最佳实践

### 1. 定期收集知识

建议在以下时机收集知识：
- 解决重要技术问题后
- 学习新技术概念后
- 发现最佳实践后
- 项目架构讨论后

### 2. 合理分类

使用一致的分类体系，便于后续检索：
- 技术知识 > 具体技术领域
- 问题解决方案 > 教程/故障排除
- 最佳实践 > 具体实践领域

### 3. 标签管理

使用清晰的标签：
- 技术标签：Git、Python、React 等
- 类型标签：教程、问题、最佳实践等
- 状态标签：已验证、待验证等

### 4. 定期维护

定期检查和更新知识库：
- 删除过时的知识
- 更新不完整的信息
- 合并重复的知识条目

## 常见问题

### Q1：为什么知识库是空的？

**A**：知识库需要手动触发收集或配置自动收集。请参考"快速开始"部分。

### Q2：如何配置自动收集？

**A**：在个人规则中添加自动收集配置，或设置环境变量 `MEMORY_KITCHEN_AUTO_COLLECT=true`。

### Q3：知识收集不准确怎么办？

**A**：可以手动编辑知识库中的文件，或改进 `knowledge_collector.py` 中的收集逻辑。

### Q4：如何备份知识库？

**A**：直接复制整个知识库目录到安全位置，或使用 Git 进行版本控制。

### Q5：知识库占用空间太大怎么办？

**A**：定期清理过时和重复的知识，或使用 Git 的历史版本管理功能。
