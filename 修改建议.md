这段代码是一个很好的起点，但在实际生产环境中，它存在几个明显的问题和局限性。为了满足你提出的“更全面、从上下文中提取隐含信息（如问题、解决方案、技术栈、工具）”的需求，我进行了深入的代码审查和重构。

1. 代码审查与问题分析
逻辑漏洞与性能问题：

lines.index(line) 性能陷阱：在 extract_question_answer_pairs 中，循环内部使用 lines.index(line) 会导致 O(N
2
 ) 的时间复杂度，且如果对话中有重复行，索引会出错。
上下文提取过于简单：extract_knowledge_from_context 仅通过标点符号分割句子，这在技术文档中很不准确（例如 URL、版本号、文件路径中常包含标点）。
硬编码依赖：直接导入 config 和 knowledge_organizer，如果这些文件不存在，脚本会直接报错崩溃，缺乏容错性。
提取能力不足：

仅限显式问答：目前的逻辑主要寻找“用户问 - 助手答”的结构。但在实际开发对话中，很多高价值信息是陈述句，例如：“我刚刚用 React 18 重构了项目，解决了 内存泄漏 问题，使用了 useMemo。”这不是问答，但极具价值。
代码块忽略：技术对话中，代码片段（Code Snippets）往往是最核心的知识，原代码未对代码块做特殊处理。
实体识别弱：对于版本号、错误码、特定工具名称的识别不够敏感。
价值判断单一：

仅通过长度和寒暄词过滤，容易漏掉短小精悍的配置命令或错误码。
2. 优化后的代码方案
以下是重构后的版本。主要改进点：

隐含知识提取：新增 extract_implicit_knowledge，专门挖掘技术栈、问题 - 解决对、决策记录。
代码块增强：识别 Markdown 代码块，将其作为高权重知识。
实体识别增强：增加对版本号、错误类型、工具链的识别。
鲁棒性提升：处理缺失的模块依赖，优化循环逻辑。
结构化输出：知识条目增加 source_type（来源类型）和 confidence（置信度）。
#!/usr/bin/env python3
"""
高级自动知识收集器 (Advanced Knowledge Collector)

此脚本用于深度分析对话内容，不仅提取显式问答，还能从上下文中
挖掘隐含的技术栈、问题解决方案、工具使用记录等高价值信息。
"""

import re
import json
import os
import uuid
from datetime import datetime
from typing import List, Dict, Any, Tuple, Optional

# --- 配置与容错处理 ---
try:
    from config import get_knowledge_base_dir
except ImportError:
    def get_knowledge_base_dir():
        return "./knowledge_base"

# --- 常量定义 ---
TECH_STACK_KEYWORDS = {
    'Languages': ['Python', 'Java', 'JavaScript', 'TypeScript', 'Go', 'Rust', 'C++', 'SQL', 'Shell'],
    'Frontend': ['React', 'Vue', 'Angular', 'Svelte', 'HTML', 'CSS', 'Webpack', 'Vite'],
    'Backend': ['Django', 'Flask', 'FastAPI', 'Spring', 'Node.js', 'Express', 'Nginx'],
    'DevOps': ['Docker', 'Kubernetes', 'Jenkins', 'GitLab CI', 'GitHub Actions', 'Terraform', 'Ansible'],
    'Database': ['MySQL', 'PostgreSQL', 'MongoDB', 'Redis', 'Elasticsearch', 'SQLite'],
    'Cloud': ['AWS', 'Azure', 'GCP', 'AliCloud', 'S3', 'EC2', 'Lambda'],
    'Tools': ['Git', 'VS Code', 'PyCharm', 'Postman', 'Jira', 'Linux', 'MacOS']
}

PROBLEM_PATTERNS = [
    r'报错', r'错误', r'Exception', r'Error', r'Failed', r'Bug', r'问题',
    r'无法', r'不能', r'崩溃', r'泄漏', r'超时', r'404', r'500', r'502'
]

SOLUTION_PATTERNS = [
    r'解决', r'修复', r'搞定', r'Workaround', r'方案', r'通过', r'使用',
    r'配置', r'更新', r'升级', r'重装', r'重启', r'修改'
]

DECISION_PATTERNS = [
    r'决定', r'选择', r'采用', r'放弃', r'计划', r'架构', r'设计', r'重构'
]

class KnowledgeCollector:
    def __init__(self):
        self.knowledge_items = []

    def collect(self, input_data: Any) -> List[Dict]:
        """主入口：处理输入并收集知识"""
        dialogue_text = self._normalize_input(input_data)

        # 1. 提取显式问答对
        qa_pairs = self._extract_qa_pairs(dialogue_text)

        # 2. 提取隐含知识 (技术栈、问题解决、决策)
        implicit_facts = self._extract_implicit_knowledge(dialogue_text)

        # 3. 提取代码块 (高价值信息)
        code_snippets = self._extract_code_snippets(dialogue_text)

        # 4. 合并并处理所有候选知识
        all_candidates = []
        for q, a in qa_pairs:
            all_candidates.append({'question': q, 'answer': a, 'type': 'QA'})
        for fact in implicit_facts:
            all_candidates.append({'question': fact['topic'], 'answer': fact['detail'], 'type': 'Fact'})
        for code in code_snippets:
            all_candidates.append({'question': f"代码示例：{code['lang']}", 'answer': code['content'], 'type': 'Code'})

        # 5. 过滤、增强并保存
        for item in all_candidates:
            if self._is_valuable(item):
                enriched_item = self._enrich_item(item)
                self.knowledge_items.append(enriched_item)

        return self.knowledge_items

    def _normalize_input(self, input_data: Any) -> str:
        """统一输入格式为字符串"""
        if isinstance(input_data, dict):
            parts = []
            if 'system' in input_data: parts.append(f"[System]: {input_data['system']}")
            if 'messages' in input_data:
                for msg in input_data['messages']:
                    role = msg.get('role', 'unknown')
                    content = msg.get('content', '')
                    parts.append(f"[{role}]: {content}")
            if 'context' in input_data: parts.append(f"[Context]: {input_data['context']}")
            return "\n".join(parts)
        return str(input_data)

    def _extract_qa_pairs(self, text: str) -> List[Tuple[str, str]]:
        """提取显式的问答对，优化了多行处理和状态机逻辑"""
        pairs = []
        lines = text.split('\n')

        current_role = None
        current_content = []
        last_question = None

        # 编译正则以提高性能
        user_re = re.compile(r'^\s*\[?(用户|User|user|U)\]?\s*[:：]')
        assistant_re = re.compile(r'^\s*\[?(助手|Assistant|assistant|A|AI)\]?\s*[:：]')

        for line in lines:
            line_stripped = line.strip()
            if not line_stripped:
                continue

            if user_re.match(line_stripped):
                # 遇到新问题，保存之前的 QA
                if last_question and current_content:
                    pairs.append((last_question, "\n".join(current_content)))
                current_content = []
                # 提取问题内容
                last_question = re.sub(r'^\s*\[?.*?\]?\s*[:：]\s*', '', line_stripped)
                current_role = 'user'

            elif assistant_re.match(line_stripped):
                # 如果是助手回答，且之前有问题
                if last_question:
                    content = re.sub(r'^\s*\[?.*?\]?\s*[:：]\s*', '', line_stripped)
                    current_content.append(content)
                current_role = 'assistant'

            elif current_role == 'assistant' and last_question:
                # 多行回答的延续
                current_content.append(line_stripped)

        # 处理最后一对
        if last_question and current_content:
            pairs.append((last_question, "\n".join(current_content)))

        return pairs

    def _extract_implicit_knowledge(self, text: str) -> List[Dict]:
        """从上下文中挖掘非问答形式的知识（技术栈、问题、决策）"""
        facts = []

        # 1. 技术栈识别
        for category, tools in TECH_STACK_KEYWORDS.items():
            for tool in tools:
                # 查找工具名称及其上下文
                pattern = re.compile(rf'.{{0,30}}{re.escape(tool)}.{{0,50}}', re.IGNORECASE)
                matches = pattern.findall(text)
                for match in matches:
                    if len(match) > 10: # 过滤太短的匹配
                        facts.append({
                            'topic': f"技术栈：{tool}",
                            'detail': match.strip(),
                            'category': category
                        })

        # 2. 问题 - 解决模式识别 (即使没有问号)
        # 逻辑：寻找包含“问题词”的句子，随后寻找包含“解决词”的句子
        sentences = re.split(r'[。！？\n]', text)
        for i, sentence in enumerate(sentences):
            sentence = sentence.strip()
            if not sentence: continue

            has_problem = any(re.search(p, sentence) for p in PROBLEM_PATTERNS)
            has_solution = any(re.search(p, sentence) for p in SOLUTION_PATTERNS)

            if has_problem:
                # 尝试在后续句子找解决方案
                solution_text = []
                for j in range(i + 1, min(i + 4, len(sentences))):
                    next_s = sentences[j].strip()
                    if any(re.search(p, next_s) for p in SOLUTION_PATTERNS):
                        solution_text.append(next_s)
                    elif next_s:
                        solution_text.append(next_s) # 假设紧接着的是解释
                    else:
                        break

                if solution_text:
                    facts.append({
                        'topic': "遇到的问题",
                        'detail': f"问题：{sentence} | 解决方案：{' '.join(solution_text)}",
                        'category': "故障排除"
                    })

        # 3. 决策与架构记录
        for sentence in sentences:
            sentence = sentence.strip()
            if any(re.search(p, sentence) for p in DECISION_PATTERNS):
                # 提取涉及技术的决策
                tech_mentioned = False
                for tools in TECH_STACK_KEYWORDS.values():
                    if any(t in sentence for t in tools):
                        tech_mentioned = True
                        break

                if tech_mentioned or len(sentence) > 15:
                    facts.append({
                        'topic': "架构决策/记录",
                        'detail': sentence,
                        'category': "项目决策"
                    })

        return facts

    def _extract_code_snippets(self, text: str) -> List[Dict]:
        """提取 Markdown 代码块"""
        snippets = []
        # 匹配 ```language ... ```
        pattern = re.compile(r'```(\w*)\n(.*?)```', re.DOTALL)
        matches = pattern.findall(text)

        for lang, code in matches:
            code = code.strip()
            if len(code) > 20: # 过滤太短的代码
                snippets.append({
                    'lang': lang if lang else 'text',
                    'content': code
                })
        return snippets

    def _is_valuable(self, item: Dict) -> bool:
        """增强版价值判断"""
        q = item.get('question', '')
        a = item.get('answer', '')
        content = q + " " + a

        # 1. 非空检查
        if not q or not a:
            return False

        # 2. 代码块直接视为高价值
        if item.get('type') == 'Code':
            return True

        # 3. 过滤寒暄
        trivial = ['你好', '谢谢', '再见', 'ok', '好的', '收到']
        if any(t in content for t in trivial) and len(content) < 30:
            return False

        # 4. 包含技术关键词或错误码视为有价值
        tech_indicators = PROBLEM_PATTERNS + SOLUTION_PATTERNS
        if any(re.search(p, content) for p in tech_indicators):
            return True

        # 5. 长度阈值
        if len(a) < 15 and item.get('type') != 'Fact':
            return False

        return True

    def _enrich_item(self, item: Dict) -> Dict:
        """添加元数据、标签和分类"""
        content = f"{item['question']} {item['answer']}"

        # 提取标签
        tags = set()
        for category, tools in TECH_STACK_KEYWORDS.items():
            for tool in tools:
                if tool.lower() in content.lower():
                    tags.add(tool)
                    tags.add(category) # 同时添加大类标签

        # 添加类型标签
        tags.add(item.get('type', 'General'))

        # 确定分类
        categories = ['未分类']
        if any(re.search(p, content) for p in PROBLEM_PATTERNS):
            categories.append('故障排除')
        if any(re.search(p, content) for p in DECISION_PATTERNS):
            categories.append('架构决策')
        if item.get('type') == 'Code':
            categories.append('代码片段')

        return {
            "id": str(uuid.uuid4()),
            "timestamp": datetime.now().isoformat(),
            "source_type": item.get('type', 'QA'),
            "question": item['question'],
            "answer": item['answer'],
            "tags": list(tags),
            "categories": list(set(categories)),
            "confidence": 0.9 if item.get('type') == 'Code' else 0.7
        }

    def save(self, output_dir: Optional[str] = None):
        """保存知识并尝试组织"""
        if output_dir is None:
            output_dir = get_knowledge_base_dir()

        os.makedirs(output_dir, exist_ok=True)
        timestamp = datetime.now().strftime('%Y%m%d_%H%M%S')
        filename = f'knowledge_dump_{timestamp}.json'
        filepath = os.path.join(output_dir, filename)

        with open(filepath, 'w', encoding='utf-8') as f:
            json.dump(self.knowledge_items, f, ensure_ascii=False, indent=2)

        print(f"✅ 已保存 {len(self.knowledge_items)} 条知识到 {filepath}")

        # 尝试调用组织器，但失败不中断
        try:
            from knowledge_organizer import organize_knowledge
            organize_knowledge(self.knowledge_items, output_dir)
        except ImportError:
            print("⚠️ 未找到 knowledge_organizer 模块，跳过自动组织步骤。")
        except Exception as e:
            print(f"⚠️ 组织知识库时出错：{e}")

3. 主要改进点详解
类结构封装 (KnowledgeCollector)：

将功能封装在类中，便于维护状态和扩展配置。
方法职责更单一（输入标准化、QA 提取、隐含提取、代码提取、价值判断、增强元数据）。
隐含知识挖掘 (_extract_implicit_knowledge)：

技术栈扫描：不再依赖问答，而是扫描全文，只要提到 "React", "Docker", "MySQL" 等关键词及其上下文，就记录为一条知识。
问题 - 解决链：即使没有问号，如果一句话包含 "报错/错误"，且后续句子包含 "解决/修复/配置"，会自动关联为一条故障排除知识。
决策记录：识别 "决定/采用/架构" 等词，记录项目中的关键决策，这对知识库的长期演进非常重要。
代码块优先 (_extract_code_snippets)：

专门的正则提取 Markdown 代码块。
代码块被标记为 type: Code，并在价值判断中获得最高优先级（因为代码通常是最干货的部分）。
提取语言类型（如 python, json），便于后续高亮显示。
鲁棒性与性能：

正则预编译：在类外或初始化时编译正则，避免循环中重复编译。
索引优化：移除了 lines.index()，改用迭代器或状态机逻辑，避免 O(N
2
 ) 复杂度。
依赖容错：config 和 knowledge_organizer 的导入都加了 try-except，确保脚本在缺少辅助模块时也能运行核心功能。
元数据增强：

source_type：区分知识是来自 QA、隐含事实还是代码，便于前端展示或后续处理。
confidence：给代码块更高的置信度，方便后续排序。
tags & categories：基于预定义的 TECH_STACK_KEYWORDS 字典自动打标，比原来的硬编码列表更易维护。
4. 使用建议
配置化：建议将 TECH_STACK_KEYWORDS 和 PATTERNS 提取到单独的 config.yaml 或 config.json 文件中，这样你可以随时添加新的技术栈（比如明天出了个新框架）而无需修改代码。
向量数据库集成：保存为 JSON 只是第一步。建议后续将 question 和 answer 的 Embedding 向量存入向量数据库（如 Chroma, Milvus），以便进行语义搜索，而不仅仅是关键词匹配。
人工审核回路：自动提取难免有误（置信度再高也不是 100%）。建议增加一个 "待审核" 状态，提取后的知识先存入临时表，由人工确认后再合并到主知识库。
隐私过滤：在 collect 之前，建议增加一步 PII（个人敏感信息）过滤，移除可能的 API Key、密码、手机号等信息，防止泄露到知识库。
